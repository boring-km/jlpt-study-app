# JLPT N2 한자 읽기 암기 앱

## 프로젝트 개요
- **목표:** JLPT N2 단어의 한자 읽기(히라가나)를 암기하는 iOS 앱
- **타겟:** 한자 읽기에 어려움을 겪는 한국어 사용자
- **디자인:** 미니멀
- **플랫폼:** iOS only (Flutter)

## 데이터 준비

### 원본 데이터
- **출처:** [open-anki-jlpt-decks](https://github.com/jamsinclair/open-anki-jlpt-decks) (MIT License)
- **파일:** `assets/data/n2_raw.csv`
- **총 단어 수:** 1,906개
- **제공 필드:** expression(한자), reading(히라가나), meaning(영어), tags, guid

### 데이터 가공 파이프라인

| 단계 | 작업 | 상태 |
|------|------|------|
| 1 | CSV 데이터셋 다운로드 | [x] |
| 2 | 한국어 뜻 + 예문 생성 (100개씩 배치) | 진행중 (1/20) |
| 3 | 전체 배치 병합 → 최종 JSON 생성 | [ ] |
| 4 | 데이터 검증 (누락/중복 체크) | [ ] |

### 최종 데이터 스키마
```json
{
  "id": 1,
  "expression": "題名",
  "reading": "だいめい",
  "meaning_en": "title",
  "meaning_ko": "제목",
  "has_kanji": true,
  "example": {
    "ja": "この本の題名は何ですか。",
    "reading": "このほんのだいめいはなんですか。",
    "ko": "이 책의 제목은 무엇입니까?"
  }
}
```

### 한국어 뜻 + 예문 생성 전략
- **방법:** 이 세션에서 LLM으로 직접 생성
- **배치:** 100개씩 20배치, 각 배치 완료 후 JSON 파일로 저장
- **예문:** 단어당 1개 (일본어 예문 + 히라가나 읽기 + 한국어 해석)

## 앱 기능 (우선순위순)

### Phase 1 - MVP
1. **플래시카드** — 한자 → 히라가나 읽기 + 한국어 뜻
2. **퀴즈** — 한자 보고 히라가나 4지선다
3. **단어장** — 전체 리스트 검색/필터

### Phase 2
4. **간격 반복 (SR)** — SM-2 알고리즘 기반 복습 스케줄
5. **학습 통계** — 진도율, 정답률

### Phase 3
6. **예문 학습** — 예문 속 한자 읽기 연습
7. **북마크** — 어려운 단어 모아보기

## 기술 스택
- **Framework:** Flutter (iOS only)
- **로컬 DB:** SQLite (sqflite) 또는 Hive
- **상태관리:** Riverpod
- **데이터:** 앱 번들 JSON (assets)
