# JLPT 앱 기획서

## 개요
- **목표:** JLPT N2/N3 단어 암기 iOS 앱
- **타겟:** 한자 읽기(히라가나)를 공부하는 한국어 사용자
- **디자인:** 미니멀

---

## 설정

| 항목 | 기본값 | 변경 |
|------|--------|------|
| 시험 날짜 | 2026.07.05 | 설정 화면에서 변경 가능 |
| 하루 학습량 | 자동 계산 | (총 단어 수 ÷ 남은 날) |

- 당일 학습을 완료하지 못한 경우 → 남은 날 기준으로 **자동 재계산**

---

## 학습 순서

- **N3 (2,046개) 전부 완료 → 자동으로 N2 (1,905개) 시작**
- 모든 화면에서 단어별 **N2 / N3 뱃지** 항상 표시

---

## 홈 화면

- D-DAY 카운터
- 오늘 진도 (예: 오늘 44개 중 12개 완료)
- 오늘 학습 시작 버튼
- 복습 바로가기
- 탐색 바로가기

---

## 오늘의 학습 흐름

```
① 플래시카드
   오늘의 단어 N개를 한 장씩 확인
   - 앞면: 한자(expression)
   - 탭하면: 히라가나 읽기 + 한국어 뜻 + 예문(일본어/한국어)
        ↓
② 퀴즈
   4지선다 — 한자를 보고 올바른 히라가나 읽기 선택
        ↓
③ 오답노트
   틀린 단어 목록 + 정답 확인
        ↓
④ 전체 세트 다시 퀴즈
        ↓
   ② ~ ④ 반복 (전부 맞을 때까지)
        ↓
⑤ 오늘 학습 완료 ✓
   (해당 단어들 "완료" 처리 → 복습 대상에 추가)
```

---

## 복습

- **대상:** 퀴즈를 전부 통과(완료)한 단어만
- **접근:** 오늘 학습과 별개로 **언제든지** 가능
- **선택:** 사용자가 직접 복습할 단어 선택
- **형태:** 퀴즈 (4지선다)

---

## 탐색 (학습 흐름과 무관, 자유롭게 접근)

| 화면 | 설명 |
|------|------|
| 단어 리스트 | 전체 단어 검색 / 필터 (N2, N3, 완료 여부 등) |
| 플래시카드 브라우저 | 전체 단어를 자유롭게 넘겨보기 |

---

## 완료 후

- N3 + N2 전체 완료 시 → **복습 전용 모드** 로 전환

---

## 데이터

| 레벨 | 단어 수 | 파일 |
|------|---------|------|
| N3 | 2,046개 | `assets/data/n3_words.json` |
| N2 | 1,905개 | `assets/data/n2_words.json` |

- 각 단어: `expression`(한자), `reading`(히라가나), `meaning_ko`(한국어 뜻), `example`(예문)

---

## 기술 스택

- **Framework:** Flutter (iOS only)
- **상태관리:** Riverpod
- **로컬 DB:** sqflite (학습 기록 저장)
- **단어 데이터:** JSON assets (메모리 로드)
- **라우팅:** go_router
