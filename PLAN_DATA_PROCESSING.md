# 데이터 가공 계획: N2 단어 한국어 번역 + 예문 생성

## 현황
- 원본: `assets/data/n2_raw.csv` (1,906개 단어)
- 필드: expression, reading, meaning(영어), tags, guid
- 한자 포함 단어: 1,608개 / 한자 없음(히라가나·카타카나): 298개

## 목표 출력
각 단어별:
- `meaning_ko`: 한국어 뜻
- `example_ja`: 일본어 예문 1개
- `example_reading`: 예문의 히라가나 읽기
- `example_ko`: 예문의 한국어 해석

## 처리 방식: 이 세션에서 직접 생성

### 배치 전략
- **100개씩 19배치** + 마지막 1배치(6개) = 총 20배치
- 각 배치 완료 후 즉시 **JSON 파일로 저장** → `assets/data/batch_01.json` ~ `batch_20.json`
- 전체 완료 후 배치 파일들을 **하나의 최종 JSON으로 병합** → `assets/data/n2_words.json`

### 안전성 분석

| 리스크 | 대응 |
|--------|------|
| 컨텍스트 압축으로 이전 대화 유실 | 배치마다 파일 저장 → 이미 저장된 데이터는 안전 |
| 세션 중단 | 저장된 배치 파일 확인 후 이어서 처리 가능 |
| 생성 품질 불일치 | 첫 배치 후 샘플 검수, 프롬프트 조정 |
| 중복/누락 | 배치 번호 + 원본 인덱스로 추적 |

### 결론: **실현 가능**
- 배치마다 파일 저장하면 컨텍스트 압축/세션 중단에도 데이터 유실 없음
- 이어서 처리 가능 (저장된 파일 개수로 진행 상황 파악)
- 추가 비용 없음

## 처리 순서

### Step 1: 파일럿 (1배치 = 100개)
- [ ] 첫 100개 단어 한국어 뜻 + 예문 생성
- [ ] `assets/data/batch_01.json`으로 저장
- [ ] 품질 검수 (사용자 확인)

### Step 2: 나머지 배치 처리
- [ ] batch_02 ~ batch_20 순차 생성 및 저장
- [ ] 각 배치 저장 후 진행률 표시

### Step 3: 병합 및 검증
- [ ] 전체 배치 파일 → `n2_words.json` 병합
- [ ] 누락/중복 검증
- [ ] furigana 빈 값 보완
